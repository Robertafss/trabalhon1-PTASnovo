API - USER

Falar as tecnologias e explicar os endpoints

Quais foram os endpoints que nós utilizamos?

1 - routes.post('/user', userController.createUser);

2 - routes.get('/users', userController.findUsers);

3 - routes.delete('/user/:id', userController.deleteUser);

4 - routes.put('/user/:id', userController.updateUser);

5 - routes.post('/user/authenticated', userController.authenticatedUser);

1 - Método HTTP: POST '/user'

Usado para criar um novo user, após enviar uma solicitação (POST), o controlador, carrega a solicitação, faz a criação de um novo user com todas as informações, e retorna todas as informações.

2 - Método HTTP: GET '/users'

Usado pra recuperar a lista criada com todos os usuários, após enviar uma solicitação (GET), o controlador faz o processamento da solicitação, retornando uma lista de users

3 - Método HTTP: DELETE '/user/:id'

Usado para excluir um determinado usuário, utilizando seu id, sendo assim, o id do user que será excluído é passado na url, o controlador faz o processamento da solicitação (DELETE), assim, encontra o user com seu respectivo id, e o deleta do sistema.

4 - Método HTTP: PUT '/user/:id'

Usado para fazer atualizações nos dados de um user já existente, utilizando também seu id, o id do user que será excluído é passado na url, o controlador faz o processamento da solicitação (PUT), faz uma busca pelo usuário com seu id, e atualiza seus respectivos dados no sistema.

5 - Método HTTP: POST '/user/authenticated'

Usado para fazer a autenticação de um user, e para que o usuário faça login no sistema, após enviar a solicitação (POST), o controlador faz o processamento da solicitação, faz a verificação de todos os dados fornecidos pelo usuário, e, se estiverem corretas, um novo token será gerado que permitirá o acesso do usuário ao sistema.
